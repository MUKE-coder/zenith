# Next.js

Zenith is built for TypeScript, and Next.js is the first-class path. `npx zenith init` detects your router and scaffolds everything; this page explains what it writes so you can do it by hand or understand what's there.

<Callout type="tip">
The fastest path is `npx zenith init` — it writes `zenith.config.js`, the dashboard route for your router, and adds the config to `.gitignore`. The rest of this page is the manual version.
</Callout>

## 1. Configure

`zenith.config.js` holds two secrets — `apiKey` and `jwtSecret` — so it's read server-side only and stays out of git:

```js title="zenith.config.js"
module.exports = {
  backendUrl: process.env.ZENITH_URL,       // your Zenith service
  siteKey:    process.env.ZENITH_SITE_KEY,   // public — ships in the snippet
  apiKey:     process.env.ZENITH_API_KEY,    // secret — server-side only
  dashboardPath: "/analytics-dashboard",
  protected: true,
  passwordHash: process.env.ZENITH_PW_HASH,  // from `npx zenith hash`
  jwtSecret:    process.env.ZENITH_JWT_SECRET,
  siteDomain:   "example.com",
};
```

## 2. Add the tracker

The tracker is cookieless and about 1&nbsp;KB. Spread `trackerScriptProps` onto a `<script>` in your root layout — no import of the package into browser code, and no React dependency:

```tsx title="app/layout.tsx"
import { trackerScriptProps } from "zenith-analytics";
import config from "../zenith.config.js";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <head>
        <script {...trackerScriptProps(config)} />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

Pageviews — including client-side route changes — are tracked automatically. For custom events, see [Tracking & events](/docs/tracking).

## 3. Mount the dashboard

This is the signature: a page on **your own domain** that shows the client's analytics, same-origin and password-gated.

### App Router

Create `app/analytics-dashboard/[[...zenith]]/route.ts`:

```ts title="app/analytics-dashboard/[[...zenith]]/route.ts"
import { createZenithRoute } from "zenith-analytics/next";
import config from "../../../zenith.config.js";

export const { GET, POST } = createZenithRoute(config);

// Without this, Next may statically render the route and serve every visitor
// the same cached page — including the password gate.
export const dynamic = "force-dynamic";
```

### Pages Router

Create `pages/api/zenith/[[...zenith]].ts`:

```ts title="pages/api/zenith/[[...zenith]].ts"
import { createZenithApiRoute } from "zenith-analytics/next";
import config from "../../../zenith.config.js";

export default createZenithApiRoute(config);

// The handler reads the password form itself; Next's parser would consume
// the stream before it could.
export const config = { api: { bodyParser: false } };
```

Then rewrite the dashboard path onto the API route in `next.config.js`:

```js title="next.config.js"
module.exports = {
  async rewrites() {
    return [
      { source: "/analytics-dashboard", destination: "/api/zenith" },
      { source: "/analytics-dashboard/:path*", destination: "/api/zenith/:path*" },
    ];
  },
};
```

<Callout type="warning">
`dynamic = "force-dynamic"` (App Router) and `bodyParser: false` (Pages Router) are not optional. The first stops Next caching the password gate; the second lets the handler read the login form. `npx zenith init` sets both.
</Callout>

## 4. Custom events

Anywhere in your client bundle:

```ts
"use client";
import { track } from "zenith-analytics/client";

export function SignupButton() {
  return <button onClick={() => track("signup", { plan: "pro" })}>Sign up</button>;
}
```

`zenith-analytics/client` is browser-safe — it holds no secrets and never throws, so a failed analytics call can't break the page.

## Entry points

| Import | Runs | Use it for |
| --- | --- | --- |
| `zenith-analytics` | Server | Config, the proxy handler, `trackerScriptProps`. Holds secrets — never import into browser code. |
| `zenith-analytics/client` | Browser | `track()`. No config, no secrets. |
| `zenith-analytics/next` | Server | The App Router and Pages Router adapters. |

Then open `yoursite.com/analytics-dashboard` and enter the password. Your client sees their analytics — on their own domain.
