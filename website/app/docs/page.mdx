# Zenith documentation

Zenith is privacy-first, multi-site web analytics and SEO auditing that you **self-host once** and share with clients — where each client's dashboard lives natively on their own domain.

You install the [`zenith-analytics`](https://www.npmjs.com/package/zenith-analytics) package in your app (it's built for TypeScript, and first-class for Next.js), point it at your Zenith service, and you're collecting cookieless pageviews in minutes.

## What you get

- **Cookieless tracking.** A tiny (~1&nbsp;KB) script and a daily-rotating visitor hash — no cookies, no fingerprinting, no consent banner.
- **A multi-site console.** Deploy once; every site you manage reports into it. Switch between clients from one dashboard.
- **Domain-native dashboards.** Your client reads their analytics at `theirsite.com/analytics-dashboard`, same-origin and password-gated.
- **On-demand SEO audits.** Headless-Chromium audits of titles, meta, links, structured data, and Core Web Vitals.
- **Automatic monthly reports.** Each client is emailed last month's numbers on the 1st.
